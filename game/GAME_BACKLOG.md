## (B) BUGS / fixes

**[B1] Enemy AI switch has no `default:` case.**
`src/combat/pit.js` (~:1525–1846). An enemy spawned with an unrecognized `type` gets no AI and stands inert — silent soft-failure rather than a thrown error. Low risk today (all spawn sites use known types) but fragile for future content. _Why it matters: future content bugs would fail silently._ Effort **S**, **P3**. (Source: code scan Task 3.)

**[B2] Evolution-choice mirror failures swallowed by empty `catch`.**
`src/combat/pit.js:185` and `:2150` silently swallow failures to mirror the evo pick into `GameState.player`. If that write throws, persisted evo state can desync from in-run state with no log. _Why it matters: data-integrity path for the one piece of cross-run player state._ Effort **S**, **P2**. (Source: code scan Task 3 — flagged AMBIGUOUS.) Other empty catches (localStorage/vibrate/voice guards) are intentional and not listed.

**[B3] Possible perf/hang under sustained combat (open lead).**

Suspected unbounded entity growth or an O(n²)/while-loop under sustained combat — not yet root-caused, and not confirmed to reproduce in real play. _Why it matters: a real long-fight softlock/perf cliff would be a progression-blocker._ Effort **M**, **P2**. Needs 5-Whys + (if real) a "frame time stays bounded under sustained combat" regression case. (Source: REGRESSION_TESTS.md.)
